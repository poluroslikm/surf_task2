## Обзор

Экран «Мои бронирования» — домашняя точка вкладки «Мои записи» таб-бара авторизованной зоны. Он
показывает клиенту список его записей на кулинарные классы, разделённый на «Предстоящие» и
«Прошедшие», с бейджем статуса, ключевыми параметрами слота (дата/время, программа, шеф) и
выбранным вариантом экипировки по каждой записи. Экран не выполняет действий записи или отмены
сам — это точка входа: тап по карточке ведёт в детали ([SCR-006](SCR-006-booking-details.md)), где
доступны отмена предстоящей записи и (для прошедшей неотменённой) переход к оценке шефа.

### User Story

> Как клиент, я хочу видеть список своих бронирований — предстоящих и прошедших,
> чтобы контролировать, куда и когда я записан, и при необходимости перейти к деталям записи.

### Бизнес-ценность

- Снимает необходимость хранить в голове или в переписке (WhatsApp/таблица), на какие классы
  клиент записан — единая точка правды по собственным записям (BR-1, BR-2).
- Даёт клиенту самостоятельный контроль над записью без обращения к владельцу студии — открывает
  путь к отмене (UC-2) и к оценке шефа после класса (FR-19, FR-20), не нагружая администратора.

## Используемые запросы

> Все API-запросы экрана — по контракту [`../api/`](../api/). Общий паттерн обработки ответа
> (Loading/Content/Empty/Error) — [00-foundations.md §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана),
> здесь не повторяется — указывается только специфика конкретного запроса.

### listBookings

**Метод:** GET
**Спецификация:** [`../api/bookings/api.yaml`](../api/bookings/api.yaml) → `listBookings`
**Триггер:** Инициализация экрана (первое открытие вкладки «Мои записи» за сессию) / pull-to-refresh

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|-----------------|----------|----------|
| — | — | — | — | Запрос не принимает параметров: список не пагинируется и не поддерживает фильтр по статусу или диапазону дат (`BookingListResponse` — плоский `items`, [`api/bookings/models.yaml`](../api/bookings/models.yaml)). Группировка «Предстоящие»/«Прошедшие» — целиком клиентская логика по `slot.start_at` и `status` (см. «Элементы экрана»). |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех, есть записи | `200`, `items` — непустой массив `BookingSummary` | Состояние Content: записи распределяются по группам «Предстоящие»/«Прошедшие» (правило — «Элементы экрана» → «Логика статуса и группировки»), по умолчанию открыта «Предстоящие». |
| Успех, записей нет | `200`, `items` — пустой массив | Состояние Empty: заглушка «У вас пока нет записей» + действие «Записаться на класс» → SCR-002. |
| Unauthorized | `code = unauthorized` (401) | Сессия невалидна — переход на [SCR-001](SCR-001-auth.md) (сквозное поведение обработки истёкшей авторизации, не специфика этого экрана). |
| Internal Error | `default`/5xx, `code = internal_error` | Состояние Error — заглушка нейтральным тоном + кнопка «Обновить» ([foundations §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана), текст — [foundations §6](../3-design-brief/00-foundations.md#6-tone-of-voice-и-общая-микрокопия)). |

## Элементы экрана

### 1. Логика статуса и группировки (вычисляемая, не самостоятельный UI-элемент)

Сопоставление статуса API (`BookingStatus`, [`api/bookings/models.yaml`](../api/bookings/models.yaml))
с отображаемым бейджем и группой списка — основа для всех карточек ниже.

| `status` (API) | `slot.start_at` относительно текущего момента | Бейдж на карточке | Группа списка |
|---|---|---|---|
| `active` | в будущем | «активна» | Предстоящие |
| `active` | в прошлом | «прошла» (вычисляется на клиенте, не отдельное значение API) | Прошедшие |
| `cancelled` | не важно | «отменена» | Прошедшие (независимо от даты начала) |
| `late_cancel` | не важно | «отменена» | Прошедшие (независимо от даты начала) |
| `cancelled_by_studio` | не важно | «Отменён студией» | Прошедшие (независимо от даты начала) |

**Сортировка внутри групп.** `listBookings` отдаёт `items` одним списком по убыванию
`slot.start_at` ([`api/bookings/api.yaml`](../api/bookings/api.yaml)). После клиентской
группировки порядок внутри каждой группы переопределяется: **«Прошедшие»** остаются по убыванию
даты начала (последние по времени — сверху, порядок API не меняется); **«Предстоящие»** клиент
пересортировывает по возрастанию (ближайшие — сверху), как и в списке слотов (FR-3) — иначе группа
отобразится в обратном, контринтуитивном порядке.

### 3. Переключатель «Предстоящие / Прошедшие»

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Сегмент «Предстоящие» | Показывает записи со статусом «активна» | `items[].status`, `items[].slot.start_at` из `listBookings` | — | Тап → показать группу «Предстоящие»; активна по умолчанию при открытии экрана |
| Сегмент «Прошедшие» | Показывает записи со статусом «прошла», «отменена», «Отменён студией» | `items[].status`, `items[].slot.start_at` из `listBookings` | — | Тап → показать группу «Прошедшие» |

- Оба сегмента всегда доступны и не скрываются, даже если одна из групп пуста (см. Empty-состояние

- Активный сегмент выделяется состоянием и текстом, не только цветом.

### 4. Карточка записи (повторяющийся элемент списка)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Дата/время начала | День недели + число + время начала класса, крупно — основной ориентир опознания записи | `slot.start_at` из `BookingSummary` | — | — |
| Программа | Название программы/меню класса | `slot.program.name` из `BookingSummary` | — | — |
| Шеф | Имя шефа, ведущего класс | `slot.chef.name` из `BookingSummary` | — | — |
| Вариант экипировки | Лейбл «Своя экипировка» / «Прокатная экипировка» | `equipment_choice` (`own`/`rental`) из `BookingSummary` | — | — |
| Цена | Зафиксированная цена записи, в рублях | `price_total` из `BookingSummary` | — | — |
| Бейдж статуса | Текст + форма (заливка/контур/иконка) — «активна» / «отменена» / «Отменён студией» / «прошла» | `status` из `BookingSummary`, преобразован по таблице в п.1 | — | — |
| Карточка целиком | Кликабельная зона всей карточки | `id` из `BookingSummary` | — | Тап → [SCR-006](SCR-006-booking-details.md) с параметром `bookingId` |

**Условия доступности:**
- Карточка кликабельна и ведёт в детали независимо от статуса записи (активна, отменена, Отменён
  студией, прошла) — полная деталь и, для «Отменён студией», причина отмены доступны только на
  SCR-006.
- Тач-зона всей карточки — не менее минимального размера интерактивного элемента
  ([foundations §3.1](../3-design-brief/00-foundations.md#31-тач-зоны-и-размеры)).

## Состояния экрана

Базовый паттерн — [00-foundations.md §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана).
Специфика этого экрана:

| Состояние | Условие | Отображение |
|-----------|---------|--------------|
| Loading | Инициализация экрана, ответ `listBookings` ещё не получен | Скелетоны карточек записи в форме будущего контента, без блокирующего спиннера |
| Content | `listBookings` вернул непустой `items` | Список карточек, разделённый на «Предстоящие» (только `status = active` с `slot.start_at` в будущем) и «Прошедшие» (см. п.1 «Элементы экрана»); по умолчанию открыта «Предстоящие» |
| Empty (нет записей вообще) | `listBookings` вернул пустой `items` | Заглушка «У вас пока нет записей» + действие «Записаться на класс» → SCR-002 |
| Empty в группе «Предстоящие» | `items` непустой, но нет ни одной записи со `status = active` и `slot.start_at` в будущем | Подсказка «Нет предстоящих записей» + действие «Записаться на класс»; группа «Прошедшие» остаётся доступной и не скрывается |
| Error | Запрос `listBookings` завершился ошибкой (сеть/5xx) | Заглушка ошибки нейтральным тоном + кнопка «Обновить»; список не отображается, пока клиент не повторит запрос |

## Критерии приёмки

### Позитивные

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** у клиента есть записи с разными статусами и датами начала, **Когда** он открывает экран «Мои бронирования», **Тогда** записи разделены на группы «Предстоящие» и «Прошедшие», по умолчанию открыта «Предстоящие», и в «Предстоящие» попадают только записи со статусом «активна» |
| AC-002 | **Дано** запись отменена клиентом или студией, а время начала класса ещё не наступило, **Когда** клиент открывает экран, **Тогда** эта запись показана в группе «Прошедшие», а не в «Предстоящие» |
| AC-003 | **Дано** запись имеет статус «активна», «отменена», «Отменён студией» или «прошла», **Когда** клиент видит её карточку, **Тогда** на карточке показан бейдж с соответствующим текстом статуса, переданным текстом и формой, а не только цветом |
| AC-004 | **Дано** у клиента есть запись на класс, **Когда** он видит карточку записи в списке, **Тогда** на карточке показаны дата/время начала, программа, шеф, выбранный вариант экипировки и цена |
| AC-005 | **Дано** запись имеет статус «Отменён студией», **Когда** клиент тапает по карточке такой записи, **Тогда** открывается [SCR-006](SCR-006-booking-details.md) с указанием причины отмены |
| AC-006 | **Дано** на экране показана карточка предстоящей записи, **Когда** клиент тапает по карточке, **Тогда** открывается [SCR-006](SCR-006-booking-details.md), где доступно действие «Отменить» |
| AC-007 | **Дано** экран в состоянии Content, **Когда** клиент выполняет жест pull-to-refresh, **Тогда** список перезапрашивается (`listBookings`) и отображает актуальные записи и статусы |
| AC-008 | **Дано** клиент подтвердил отмену записи на [BS-003](BS-003-cancel-confirm.md), **Когда** отмена выполнена успешно, **Тогда** клиент возвращается на [SCR-006](SCR-006-booking-details.md) (с обновлённым статусом и снек-итогом отмены), а на SCR-005 попадает только последующим отдельным тапом «Назад» с SCR-006 |

### Негативные

| ID | Критерий |
|----|----------|
| AC-N01 | **Дано** в системе есть записи разных клиентов, **Когда** клиент открывает «Мои бронирования», **Тогда** он видит исключительно собственные записи, и на экране нет чужих или административных данных |
| AC-N02 | **Дано** клиент подтвердил отмену записи на [BS-003](BS-003-cancel-confirm.md), **Когда** отмена выполнена, **Тогда** прямого автоматического перехода со шторки сразу на SCR-005 не происходит — промежуточный экран SCR-006 не пропускается |
| AC-N03 | **Дано** запрос списка бронирований завершился ошибкой, **Когда** клиент открывает экран, **Тогда** показана заглушка ошибки нейтральным тоном с кнопкой «Обновить», и список не отображается, пока клиент не повторит запрос |

### Граничные

| ID | Критерий |
|----|----------|
| AC-E01 | **Дано** у клиента нет ни одной записи, **Когда** он открывает экран, **Тогда** показана заглушка «У вас пока нет записей» с действием «Записаться на класс», ведущим к [SCR-002](SCR-002-slot-list.md) |
| AC-E02 | **Дано** у клиента есть только прошедшие и/или отменённые записи, **Когда** он открывает группу «Предстоящие», **Тогда** показана подсказка «Нет предстоящих записей» с действием «Записаться на класс», а группа «Прошедшие» остаётся доступной и не скрывается |
| AC-E03 | **Дано** сессия клиента истекла (ответ `listBookings` — `401 unauthorized`), **Когда** клиент открывает экран, **Тогда** происходит переход на [SCR-001](SCR-001-auth.md) вместо отображения пустого или ошибочного списка |
