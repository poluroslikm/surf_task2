# Индекс переиспользуемых логик

> Бизнес- и UI-логика, общая для нескольких экранов или самодостаточная по объёму, чтобы не
> повторять её в каждом экранном ТЗ. Экраны подключают логику через раздел «Применяемые логики»
> и ссылаются сюда, не копируя описание.

| ID | Логика | Используется в | Документ |
|----|--------|-----------------|----------|
| LOGIC-001 | Авторизация по email и паролю | [SCR-001](../SCR-001-auth.md) | [LOGIC-001_Авторизация.md](LOGIC-001_Авторизация.md) |
| LOGIC-002 | Правило отмены (24 часа) | [SCR-006](../SCR-006-booking-details.md), [BS-003](../BS-003-cancel-confirm.md) | [LOGIC-002_Правило-отмены-24-часа.md](LOGIC-002_Правило-отмены-24-часа.md) |
| LOGIC-003 | Фильтрация слотов по датам | [SCR-002](../SCR-002-slot-list.md), [BS-001](../BS-001-date-filter.md) | [LOGIC-003_Фильтрация-слотов.md](LOGIC-003_Фильтрация-слотов.md) |
| LOGIC-004 | Запрос push-разрешения | [BS-002](../BS-002-booking-success.md) | [LOGIC-004_Запрос-push-разрешения.md](LOGIC-004_Запрос-push-разрешения.md) |
| LOGIC-005 | Паттерн состояний экрана | Все экраны с запросами к API | [LOGIC-005_Паттерн-состояний-экрана.md](LOGIC-005_Паттерн-состояний-экрана.md) |

Не вынесены отдельной логикой (по принципу «не изобретай сверх требований»):
- **Расчёт доступности мест** — тривиальная проверка `free_seats > 0`, нет второго независимого
  лимита (прокатный фонд не ограничен, FR-9) — не набирает сложности, достаточной для отдельного
  документа; описана инлайн в `SCR-004`.
- **Расчёт цены** — плоское значение `slot.price` без формулы (нет тарифа проката в контракте,
  `api/README.md` → «Открытые вопросы»); отдельная логика не нужна.
