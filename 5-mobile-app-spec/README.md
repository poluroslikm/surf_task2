## Соглашения

- **Платформа:** веб-приложение (PWA), открывается в браузере на телефоне, без публикации в
  App Store / Google Play (NFR-3). Только русский язык интерфейса (NFR-4).
- **API:** все запросы REST, спецификации — [`../api/`](../api/) (домены `auth`, `slots`,
  `bookings`). В ТЗ указываются точные `operationId`, метод и путь.
- **Числа не хардкодятся:** вместимость слота, цена — приходят из данных слота/API
  (`00-foundations.md` §2, принцип P3).
- **Визуальный макет** не зафиксирован Figma-файлом (бренд не задан заказчиком, см.
  `00-foundations.md` — «Объём визуальных требований»); источник структурных решений —
  дизайн-брифы в [`../3-design-brief/`](../3-design-brief/), включая ASCII-wireframe в каждом
  документе. Отдельного `design-review.md` в этом проекте нет — эквивалентная функция (проверка
  на противоречия и невыдуманность) выполнена ревью самого `3-design-brief/` по всем 11
  документам (найденные и исправленные расхождения — см. историю правок этих файлов).
- **Одна запись — одно место.** Сквозное допущение по всему ТЗ (см. `feature-list.md` §2) —
  ни один экран не содержит счётчика количества мест.
