## Обзор

Корневая вкладка таб-бара («Классы») и стартовый экран авторизованной зоны — список предстоящих
кулинарных классов (слотов) на ближайшие 7 дней от текущего момента по умолчанию, с возможностью
расширить период через фильтр дат. Экран открывает цепочку записи (список → карточка слота →
оформление → подтверждение, ≤ 3 экрана до подтверждения, NFR-2) и является первой точкой, где
клиент видит доступные, заполненные и отменённые студией классы, не подменяя и не пересчитывая
данные, полученные из API (FR-12).

### User Story

> Как клиент, я хочу видеть список предстоящих кулинарных классов на ближайшую неделю и при
> необходимости расширять период просмотра,
> чтобы быстро понять, на какой класс есть смысл записаться (дата, свободные места, программа,
> шеф), и перейти к записи.

### Бизнес-ценность

- Заменяет ручную запись через WhatsApp и Google-таблицу самостоятельным просмотром актуального
  расписания клиентом, устраняя путаницу и двойные записи (BR-1, BR-2).
- Первый экран в цепочке записи — от него напрямую зависит конверсия в запись (NFR-2: путь до
  подтверждения ≤ 3 экрана, SCR-002 первый в цепочке).
- Прозрачное отображение заполненных и отменённых студией слотов снижает число обращений клиента
  к студии за уточнением «остались ли места» или «класс точно состоится».

## Используемые запросы

> Все API-запросы экрана — по контракту [`../api/`](../api/). Общий паттерн обработки ответа
> (Loading/Content/Empty/Error) — [LOGIC-005](09_Логики/LOGIC-005_Паттерн-состояний-экрана.md),
> здесь не повторяется — указывается только специфика конкретного запроса.

### listSlots

**Метод:** GET
**Спецификация:** [`../api/slots/api.yaml`](../api/slots/api.yaml) → `listSlots`
**Триггер:** Инициализация экрана (открытие вкладки «Классы»); повтор — при применении/сбросе
фильтра ([BS-001](BS-001-date-filter.md)), pull-to-refresh, тап «Обновить» в состоянии Error, тап
«Выбрать другой период» или «Сбросить фильтр» в состоянии Empty.

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|-----------------|----------|----------|
| `date_from` | string (date-time) | Нет | `dateFilter.date_from`, если задан; иначе не передаётся — сервер по умолчанию подставляет текущий момент (LOGIC-003) | Нижняя граница периода, включительна |
| `date_to` | string (date-time) | Нет | `dateFilter.date_to`, если задан; иначе не передаётся — сервер по умолчанию подставляет `date_from + 7 дней` (LOGIC-003) | Верхняя граница периода, включительна |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех, есть слоты | `200`, `items` не пуст | Content: список карточек слотов в порядке, полученном от сервера (сортировка по `start_at` по возрастанию — на клиенте не пересчитывается), включая слоты с `free_seats = 0` (бейдж «Мест нет») и `status = cancelled` (бейдж «Отменён студией») |
| Успех, слотов нет | `200`, `items = []` | Empty state — см. «Состояния экрана» |
| Не авторизован | `code = unauthorized` (`401`) | Сквозная обработка сессии — переход на [SCR-001](SCR-001-auth.md) (не специфично для экрана) |
| Сбой при первичной загрузке | `default` (5xx) / сетевая ошибка, контент экрана ещё не показан | Error-заглушка с кнопкой «Обновить» — см. «Состояния экрана» |
| Сбой при pull-to-refresh | `default` (5xx) / сетевая ошибка, контент уже показан | Снек «Не удалось загрузить. Проверьте соединение и попробуйте снова.» (тот же текст, что при первичной загрузке — см. [LOGIC-005](09_Логики/LOGIC-005_Паттерн-состояний-экрана.md)), ранее показанный список остаётся без изменений |

## Элементы экрана

> **Колонка «Валидация»** — для полей ввода правило и текст ошибки, для остального — «—».
> **Условия доступности** — для интерактивных элементов указываются после таблицы блока.

### 1. Хедер (фикс.)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Заголовок «Классы» | Идентифицирует раздел, корневая вкладка | Статический текст | — | — |
| Иконка фильтра | Точка входа в фильтр периода; помечается индикатором (точкой), когда `dateFilter` отличается от дефолта | `dateFilter` (состояние экрана) | — | Тап → открывает [BS-001](BS-001-date-filter.md) |
| Иконка аккаунта | Точка входа в управление аккаунтом | — | — | Тап → открывает [BS-004](BS-004-account.md) |

**Условия доступности:**
- Иконка фильтра и иконка аккаунта активны всегда, независимо от состояния экрана (Loading/Content/Empty/Error).

### 2. Чип активного фильтра (условный блок)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Чип периода | Компактная строка под хедером с текстом диапазона дат, например «Период: 3–20 июля» | `dateFilter.date_from`, `dateFilter.date_to` | — | — |
| Кнопка ✕ на чипе | Мгновенный сброс периода к дефолтным 7 дням без открытия BS-001 | — | — | Тап → `dateFilter = null`, повторный запрос `listSlots` без `date_from`/`date_to` |

**Условия доступности:**
- Блок отображается только если `dateFilter` задан и отличается от дефолтного периода `[текущий момент, текущий момент + 7 дней]` (LOGIC-003).

### 3. Карточка слота (список)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Дата/время начала | День недели, число месяца, время в локальном времени студии, например «Чт, 3 июля · 18:00» | `start_at` из `listSlots` | — | — |
| Название программы | Название класса | `program.name` из `listSlots` | — | — |
| Тег уровня сложности | Отдельная визуальная метка, не часть текста названия | `program.difficulty` из `listSlots` (`novice` / `experienced`) | — | — |
| Шеф | Имя шефа, ведущего класс | `chef.name` из `listSlots` | — | — |
| Свободно / всего мест | Число свободных мест из общего числа, например «Свободно: 4 из 8» | `free_seats`, `total_seats` из `listSlots` | — | — |
| Бейдж «Мест нет» | Заменяет число свободных мест; состояние читается по тексту и иконке, не только по цвету | Производится из `free_seats = 0` при `status = scheduled` | — | — |
| Бейдж «Отменён студией» | Заменяет число свободных мест; визуально отличен от бейджа «Мест нет» (иная иконка/текст) | Производится из `status = cancelled` | — | — |
| Карточка (тач-зона) | Вся площадь карточки — единая тач-зона (≥ 44–48 pt) | `id` из `listSlots` | — | Тап → переход на [SCR-003](SCR-003-slot-card.md) с параметром `slotId` |

**Условия доступности:**
- Секция «Свободно / всего мест» показывается, если `status = scheduled` и `free_seats > 0`.
- Бейдж «Мест нет» показывается, если `status = scheduled` и `free_seats = 0`.
- Бейдж «Отменён студией» показывается, если `status = cancelled` — имеет приоритет над бейджем
  «Мест нет»: если слот отменён, значение `free_seats` не отображается вовсе, независимо от его
  числового значения.
- Карточка кликабельна для всех трёх состояний (свободные места, «Мест нет», «Отменён студией») —
  тап открывает SCR-003 в режиме просмотра; на карточке в списке нет CTA записи ни для одного из
  состояний (FR-6a, FR-18).

### 4. Empty / Error — действия

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Кнопка «Выбрать другой период» | Предлагает открыть фильтр дат | — | — | Тап → открывает [BS-001](BS-001-date-filter.md) |
| Кнопка «Сбросить фильтр» | Возврат к дефолтному периоду 7 дней | — | — | Тап → `dateFilter = null`, повторный запрос `listSlots` |
| Кнопка «Обновить» | Повтор запроса после сбоя первичной загрузки | — | — | Тап → повторный запрос `listSlots` с теми же параметрами периода |

**Условия доступности:**
- «Выбрать другой период» показывается в Empty, только если активен дефолтный период (`dateFilter = null`).
- «Сбросить фильтр» показывается в Empty, только если активен нестандартный период (`dateFilter` задан).
- «Обновить» показывается только в состоянии Error (сбой первичной загрузки), не при ошибке pull-to-refresh.

## Состояния экрана

Базовый паттерн — [LOGIC-005 Паттерн состояний экрана](09_Логики/LOGIC-005_Паттерн-состояний-экрана.md).
Специфика этого экрана:

| Состояние | Условие | Отображение |
|-----------|---------|--------------|
| Loading | Первичная загрузка `listSlots` — при открытии экрана или сразу после применения/сброса периода фильтра | Скелетон из 4–6 карточек-заглушек в форме карточки слота (не пустой экран); ориентир отклика — NFR-5 (< 2–3 с) |
| Content | `200`, `items` не пуст | Список карточек слотов за активный период (дефолт — закрытый интервал `[текущий момент, текущий момент + 7 дней]`, либо период, заданный в BS-001), отсортированный по времени начала; включает заполненные слоты с бейджем «Мест нет» и слоты, отменённые студией, с бейджем «Отменён студией» |
| Empty | `200`, `items = []` — ни одного слота в активном периоде | Текст «Пока нет доступных классов» (FR-5) + кнопка «Выбрать другой период» (если активен дефолтный период) или «Сбросить фильтр» (если активен нестандартный период) |
| Error | Сетевой сбой или ошибка сервера при первичной загрузке (или сразу после смены периода) | Заглушка ошибки с текстом «Не удалось загрузить. Проверьте соединение и попробуйте снова.» + кнопка «Обновить» |
| Обновление (pull-to-refresh) — ошибка | Сетевой сбой при обновлении потягиванием вниз, контент уже отображён | Ранее показанный список остаётся на экране без изменений; ошибка сообщается снеком «Не удалось загрузить. Проверьте соединение и попробуйте снова.» (тот же текст, что и в состоянии Error), не Error-заглушкой |

Индикатор активного фильтра (иконка-точка в хедере, чип периода) — не отдельное состояние экрана,
а элемент, накладывающийся на состояние Content/Empty/Error в зависимости от того, задан ли
нестандартный период.

## Критерии приёмки

### Позитивные

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент открывает вкладку «Классы» без ранее заданного фильтра, **Когда** экран загружает список, **Тогда** показываются слоты за закрытый интервал `[текущий момент, текущий момент + 7 дней]`, отсортированные по времени начала (ближайшие выше), и каждая карточка содержит дату/время, программу, уровень сложности, шефа, всего/свободно мест |
| AC-002 | **Дано** в списке есть слот со свободными местами (`status = scheduled`, `free_seats > 0`), **Когда** экран отображает его карточку, **Тогда** показаны дата/время начала, программа, тег уровня сложности, шеф, число свободных и общее число мест, взятые из ответа `listSlots` без пересчёта на клиенте |
| AC-003 | **Дано** в выбранном периоде есть слот с `free_seats = 0` и `status = scheduled`, **Когда** экран отображает список, **Тогда** слот остаётся на своей позиции по времени начала, вместо числа свободных мест показан бейдж «Мест нет», карточка кликабельна и открывает SCR-003 в режиме просмотра, а элемента записи на карточке нет |
| AC-004 | **Дано** в выбранном периоде есть слот со `status = cancelled`, **Когда** экран отображает список, **Тогда** слот остаётся на своей позиции по времени начала, вместо числа свободных мест показан бейдж «Отменён студией» (визуально отличный от «Мест нет»), карточка кликабельна и открывает SCR-003 в режиме просмотра, а запись на этот слот невозможна ни в списке, ни на SCR-003, ни на SCR-004 (FR-18) |
| AC-005 | **Дано** клиент открывает [BS-001](BS-001-date-filter.md) и выбирает период длиннее 7 дней, **Когда** фильтр применяется, **Тогда** список обновляется по выбранному периоду (`date_from`/`date_to` передаются в `listSlots`), и под хедером появляется чип с обозначением активного периода |
| AC-006 | **Дано** активен нестандартный период (`dateFilter` задан), **Когда** клиент нажимает ✕ на чипе фильтра, **Тогда** список возвращается к дефолтным ближайшим 7 дням (`listSlots` вызывается без `date_from`/`date_to`), и чип фильтра исчезает |

### Негативные

| ID | Критерий |
|----|----------|
| AC-N01 | **Дано** выбранный период (дефолтный или заданный фильтром) не содержит ни одного слота (`items = []`), **Когда** ответ `listSlots` получен, **Тогда** показывается Empty state с текстом «Пока нет доступных классов», и предлагается действие — «Выбрать другой период» при дефолтном периоде или «Сбросить фильтр» при активном нестандартном периоде |
| AC-N02 | **Дано** клиент впервые открывает вкладку «Классы», **Когда** запрос `listSlots` завершается сетевой ошибкой или ошибкой сервера, **Тогда** показывается Error-заглушка с текстом «Не удалось загрузить. Проверьте соединение и попробуйте снова.» и кнопкой «Обновить», повторный тап на которую повторяет запрос за тот же период |
| AC-N03 | **Дано** список уже отображает данные, **Когда** клиент тянет список вниз для обновления и запрос `listSlots` завершается ошибкой, **Тогда** показывается снек «Не удалось загрузить. Проверьте соединение и попробуйте снова.», а ранее показанный список остаётся на экране без изменений |

### Граничные

| ID | Критерий |
|----|----------|
| AC-E01 | **Дано** слот стартует ровно в момент верхней границы интервала «текущий момент + 7 дней», **Когда** экран загружает список с дефолтным периодом, **Тогда** слот включён в выдачу наравне с более ранними слотами — граница включена (закрытый интервал `[текущий момент, текущий момент + 7 дней]`) |
| AC-E02 | **Дано** слот имеет `free_seats = 0` и `total_seats > 0` при `status = scheduled`, **Когда** экран отображает его карточку, **Тогда** показан бейдж «Мест нет» вместо числа «0», и слот не скрывается из списка (FR-6a) |
| AC-E03 | **Дано** слот имеет `free_seats = total_seats` (класс полностью свободен) при `status = scheduled`, **Когда** экран отображает его карточку, **Тогда** показаны оба числа как есть, без бейджа «Мест нет» |
| AC-E04 | **Дано** слот имеет `status = cancelled` и при этом произвольное значение `free_seats` (в том числе `free_seats > 0`), **Когда** экран отображает его карточку, **Тогда** показывается только бейдж «Отменён студией» — число свободных мест не отображается ни в каком виде |
