## Обзор

Экран собирает необязательный отзыв клиента о прошедшем классе: оценку 1–5 звёзд и опциональный
текстовый комментарий. Это единственное действие экрана — форма ввода, без вывода уже
накопленных оценок шефа кому-либо: публичного рейтинга шефа для клиентов в приложении нет, а
редактирование уже отправленной оценки не предусмотрено. Экран — вложенный шаг потока «Мои
бронирования» → «Детали брони», открывается только для прошедшей неотменённой записи, по которой
оценка ещё не поставлена.

### User Story

> Как клиент, я хочу оценить шефа после посещения класса,
> чтобы поделиться впечатлением о качестве проведения (US-13).

### Бизнес-ценность

- Даёт владельцу студии сигнал о том, «кто из шефов как принимается клиентами» (BR-5), через
  существующую вне приложения инфраструктуру — без затрат на отдельную аналитическую витрину.
- Повышает вовлечённость клиента после класса лёгким необязательным действием, не создавая
  давления и не блокируя ни один другой сценарий (FR-19: оценка не обязательна).

## Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|---------------------|----------|
| `bookingId` | Навигационный параметр | UUID | Идентификатор записи, для которой ставится оценка; используется как path-параметр `submitRating` и (в резервном сценарии) `getBooking`. |
| `programTitle` | Навигационный параметр | Строка | Название программы класса — для заголовка контекста; передаётся с SCR-006 без отдельного запроса (P7). |
| `startAt` | Навигационный параметр | ISO date-time | Дата/время начала прошедшего класса — для заголовка контекста. |
| `chefName` | Навигационный параметр | Строка | Имя шефа — персонализирует вопрос «Как вам занятие с шефом [имя]?». |
| `rating` | Состояние (локальное) | `null`, `1`…`5` | Текущий выбор звёзд в форме; `null` до первого тапа по звезде. |
| `comment` | Состояние (локальное) | Свободный текст, включая пустую строку | Текущее значение поля комментария; необязательно. |

> Если экран открыт напрямую (без перехода с SCR-006 — например, после перезагрузки PWA по
> сохранённому URL), `programTitle`/`startAt`/`chefName` отсутствуют и восстанавливаются
> резервным запросом `getBooking` (см. [«Используемые запросы»](#используемые-запросы)).

## Используемые запросы

> Все API-запросы экрана — по контракту [`../api/`](../api/). Общий паттерн обработки ответа
> (Loading/Content/Empty/Error) — [00-foundations.md §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана),
> здесь не повторяется — указывается только специфика конкретного запроса.

### submitRating

**Метод:** POST `/bookings/{bookingId}/rating`
**Спецификация:** [`../api/bookings/api.yaml`](../api/bookings/api.yaml) → `submitRating`
**Триггер:** Тап на кнопку «Отправить оценку» (доступен только когда `rating != null`)

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|-----------------|----------|----------|
| `bookingId` | string (uuid), path | Да | Навигационный параметр экрана | Запись, к которой относится оценка. |
| `stars` | integer (1–5), body | Да | Состояние `rating` (звёздный рейтинг) | Целочисленная оценка 1–5, выбранная тапом по звезде. |
| `comment` | string, nullable, body | Нет | Состояние `comment` (поле комментария) | Пустая строка или отсутствие поля — валидное значение. |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | `200 Booking` с заполненным `rating` | Переход на SCR-006 со снеком «Спасибо за оценку!» ([00-foundations.md §6.1](../3-design-brief/00-foundations.md#61-каталог-снеков-успеха)). |
| Оценка уже поставлена | `409`, `code = already_rated` | Инлайн-сообщение/снек «Вы уже оценили этот класс. Изменить оценку нельзя.»; форма скрывается, экран переходит в guard-состояние «Уже оценено» (UC-4 E2). |
| Запись не подходит под условия | `422`, `code = not_ratable` | Сообщение «Оценка недоступна: запись отменена или класс ещё не начался.»; форма скрывается, экран переходит в guard-состояние «Недоступно» (UC-4 E1). |
| Токен истёк | `401` | Переход на вход (SCR-001), по сквозному правилу обработки `401`. |
| Сеть/сервер/таймаут | `5xx` / нет ответа | Снек «Не удалось выполнить. Проверьте соединение и повторите.» ([00-foundations.md §6](../3-design-brief/00-foundations.md#6-tone-of-voice-и-общая-микрокопия)); введённые `rating` и `comment` сохраняются на экране, CTA возвращается в активное состояние для повтора (UC-4 E3). |

### getBooking (резервный сценарий)

**Метод:** GET `/bookings/{bookingId}`
**Спецификация:** [`../api/bookings/api.yaml`](../api/bookings/api.yaml) → `getBooking`
**Триггер:** Инициализация экрана — только если `programTitle`/`startAt`/`chefName` не переданы навигацией (прямой заход по URL, например после перезагрузки PWA); не часть основного потока с SCR-006.

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|-----------------|----------|----------|
| `bookingId` | string (uuid), path | Да | Навигационный параметр экрана | Запись, контекст которой требуется восстановить. |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | `200 Booking` | Контент экрана заполняется программой/датой/шефом из `booking.slot`; если `booking.rating` уже заполнен — сразу показывается guard-состояние «Уже оценено»; если запись не прошла или отменена — guard-состояние «Недоступно». |
| Не найдена / чужая запись | `403` / `404` | Полноэкранная заглушка ошибки, кнопка «Обновить» ([00-foundations.md §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана)). |
| Токен истёк | `401` | Переход на вход (SCR-001). |
| Сеть/сервер/таймаут | `5xx` / нет ответа | Полноэкранная заглушка ошибки «Не удалось загрузить. Проверьте соединение и попробуйте снова.» + кнопка «Обновить». |

## Элементы экрана

### 1. Хедер (фикс.)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Кнопка «Назад» | Иконка возврата | — | — | Переход на SCR-006 без снека (отказ от оценки, UC-4 A1) |
| Заголовок | Текст «Оценка шефа» | Статический текст ([00-foundations.md §9](../3-design-brief/00-foundations.md#9-глоссарий-дизайн-специфичный-срез)) | — | — |

**Условия доступности:**
- Кнопка «Назад» активна всегда, кроме состояния «Отправка» (см. [«Состояния экрана»](#состояния-экрана)).

### 2. Контекст класса (read-only)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Название программы | Что за класс оценивает клиент | `programTitle` (навигационный параметр либо `booking.slot.program.name` из `getBooking`) | — | — |
| Дата/время начала | Уточняет, какой именно прошедший класс имеется в виду | `startAt` (навигационный параметр либо `booking.slot.start_at`) | — | — |
| Имя шефа | Персонализирует вопрос-заголовок звёздного блока | `chefName` (навигационный параметр либо `booking.slot.chef.name`) | — | — |

### 3. Звёздный рейтинг

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Вопрос-заголовок | «Как вам занятие с шефом [имя]?» | `chefName` | — | — |
| Ряд из 5 звёзд | Тач-цели ≥ 44–48 pt; тап по n-й звезде заполняет звёзды 1…n | Локальное состояние `rating` | Обязательно для отправки: пока `rating = null`, CTA «Отправить оценку» неактивен; отдельного текста ошибки нет — вопрос-заголовок и незаполненные звёзды сами объясняют, чего не хватает | Тап по звезде n → `rating = n` |
| Текстовый дубль состояния | «Оценка: не выбрана» / «Оценка: N из 5» | Локальное состояние `rating` | — | — |

**Условия доступности:**
- Ряд звёзд активен всегда в состоянии Content, кроме состояния «Отправка» (поля блокируются от изменения).
- Повторный тап по уже выбранной крайней звезде не сбрасывает `rating` в `null` — оценку можно только заменить другим значением или полностью отказаться уходом с экрана.

### 4. Поле комментария

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Textarea «Комментарий (необязательно)» | Многострочное поле свободного текста | Локальное состояние `comment` | Не обязательно; пустая строка — валидное значение; жёсткий лимит длины требованиями не задан | Ввод текста → обновление `comment` |

**Условия доступности:**
- Поле активно всегда в состоянии Content, кроме состояния «Отправка».

### 5. Нижний CTA (фикс.)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Кнопка «Отправить оценку» | Во всю ширину контентной области | Локальное состояние `rating`, `comment` | Активна только при `rating != null`; наличие/отсутствие `comment` на доступность не влияет | Тап → `POST submitRating {bookingId, stars: rating, comment}` |

**Условия доступности:**
- Disabled, пока `rating = null`.
- Enabled, как только выбрана хотя бы 1 звезда.
- В состоянии «Отправка» — индикатор загрузки, повторный тап заблокирован (защита от двойной отправки).

### 6. Guard-блоки (без формы)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|------------------|-----------|----------|
| Пояснение «Недоступно» | «Оценка недоступна: запись отменена или класс ещё не начался.» | Ответ `422 not_ratable` (`submitRating`) либо результат `getBooking` в резервном сценарии | — | — |
| Пояснение «Уже оценено» | «Вы уже оценили этот класс. Изменить оценку нельзя.» | Ответ `409 already_rated` (`submitRating`) либо `booking.rating != null` из `getBooking` | — | — |
| Кнопка «Назад» (в guard-блоке) | Единственное доступное действие в этих состояниях | — | — | Переход на SCR-006 |

## Состояния экрана

Базовый паттерн — [00-foundations.md §5](../3-design-brief/00-foundations.md#5-сквозной-паттерн-состояний-экрана).
Специфика этого экрана:

| Состояние | Условие | Отображение |
|-----------|---------|--------------|
| Loading | Только резервный сценарий: экран открыт напрямую, контекст ещё не получен от `getBooking` | Скелетон в форме блоков контекста и формы (не пустой экран) |
| Content | Контекст класса получен (передан с SCR-006 либо загружен `getBooking`), запись прошедшая и неотменённая, оценка ещё не поставлена | Полная форма: контекст класса, звёзды, комментарий, активный/неактивный по правилу CTA |
| Empty | Не применимо — экран всегда открывается для одной конкретной записи, у которой либо есть право на оценку, либо нет (см. guard-состояния ниже); пустого списка/контента здесь не бывает | — |
| Error | Резервный сценарий: сетевая/серверная ошибка при `getBooking` | Заглушка ошибки + кнопка «Обновить» |
| Недоступно (guard, UC-4 E1) | Запись отменена или класс ещё не начался — обнаружено `getBooking` (резервный сценарий) либо ответом `422 not_ratable` при попытке отправки в обход скрытой на SCR-006 кнопки | Форма (звёзды, комментарий, CTA) не отображается; только пояснение и кнопка «Назад» |
| Уже оценено (guard, UC-4 E2) | Оценка по записи уже сохранена — обнаружено `getBooking` (резервный сценарий) либо ответом `409 already_rated` | Форма не отображается; только пояснение и кнопка «Назад» |
| Отправка (submitting) | После тапа «Отправить оценку», до получения ответа `submitRating` | Индикатор загрузки на CTA; поля формы и кнопка «Назад» заблокированы от повторного действия |
| Ошибка отправки (UC-4 E3) | Сетевой сбой при `submitRating` | Снек «Не удалось выполнить. Проверьте соединение и повторите.»; `rating` и `comment` сохраняются на экране; CTA снова активен для повтора |

## Действия пользователя

| Действие | Элемент | Триггер | Результат |
|----------|---------|---------|-----------|
| Выбор оценки | Ряд звёзд | Tap по n-й звезде | `rating = n`; текстовый дубль обновляется на «Оценка: N из 5»; CTA «Отправить оценку» становится активным |

| Выход из guard-состояния | Кнопка «Назад» (в блоке «Недоступно»/«Уже оценено») | Tap | Переход на SCR-006 |
| Повтор загрузки контекста | Кнопка «Обновить» (резервный сценарий, состояние Error) | Tap | Повторный запрос `getBooking` |

## Связанные требования

| Категория | Идентификаторы |
|-----------|-----------------|
| **FR** | FR-19 (оценка 1–5 звёзд + опциональный комментарий, необязательна), FR-20 (только для прошедших неотменённых записей) |

| **US** | US-13 |

## Критерии приёмки

### Позитивные

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент на экране оценки прошедшего неотменённого класса, **Когда** он выбирает 5 звёзд, вводит текст комментария и нажимает «Отправить оценку», **Тогда** оценка сохраняется (`submitRating` → 200), связанная с записью и шефом, и происходит переход на SCR-006 со снеком «Спасибо за оценку!» |
| AC-002 | **Дано** клиент выбрал 3 звезды и оставил поле комментария пустым, **Когда** он нажимает «Отправить оценку», **Тогда** оценка сохраняется без текстового комментария (UC-4 A2), и происходит переход на SCR-006 со снеком «Спасибо за оценку!» |
| AC-003 | **Дано** клиент на экране оценки, ни одна звезда не выбрана, **Когда** он нажимает «Назад» в хедере, не отправляя форму, **Тогда** происходит переход на SCR-006 без снека, запись остаётся без оценки, и повторная попытка оценить эту запись остаётся доступной (UC-4 A1) |

### Негативные

| ID | Критерий |
|----|----------|
| AC-N01 | **Дано** запись была отменена клиентом/студией или класс ещё не начался, **Когда** клиент попадает на экран оценки (в обход скрытой на SCR-006 кнопки) и/или сервер отвечает `422 not_ratable`, **Тогда** показывается «Оценка недоступна: запись отменена или класс ещё не начался.», и форма ввода оценки не отображается (UC-4 E1) |
| AC-N02 | **Дано** оценка для этой записи уже была сохранена ранее, **Когда** клиент попадает на экран оценки этой же записи и/или сервер отвечает `409 already_rated`, **Тогда** показывается «Вы уже оценили этот класс. Изменить оценку нельзя.», форма ввода и кнопка отправки не отображаются, и повторная отправка оценки невозможна (UC-4 E2) |
| AC-N03 | **Дано** клиент выбрал звёзды и нажал «Отправить оценку», **Когда** сохранение оценки завершается сетевой ошибкой, **Тогда** показывается «Не удалось выполнить. Проверьте соединение и повторите.», выбранные звёзды и введённый комментарий сохраняются на экране, и кнопка «Отправить оценку» снова активна для повтора (UC-4 E3) |
| AC-N04 | **Дано** CTA в состоянии «Отправка» после первого тапа, **Когда** клиент повторно тапает «Отправить оценку», **Тогда** второй запрос `submitRating` не отправляется |

### Граничные

| ID | Критерий |
|----|----------|
| AC-E01 | **Дано** клиент ничего не выбрал в звёздном рейтинге, **Тогда** кнопка «Отправить оценку» неактивна независимо от текста, введённого в комментарии |
| AC-E02 | **Дано** клиент выбирает 1 звезду, **Тогда** `stars` сохраняется равным 1 (не блокируется как «низкая оценка»); **Дано** клиент выбирает 5 звёзд, **Тогда** `stars` сохраняется равным 5 |
