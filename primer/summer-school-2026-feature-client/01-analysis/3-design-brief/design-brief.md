## Цель и контекст

**«Волна»** заменяет ручную запись на SUP-прогулки через WhatsApp и бумажную тетрадь,
устраняя двойные брони и путаницу с местами. Клиент самостоятельно просматривает слоты,
фильтрует, записывается (себя и гостей), выбирает вариант доски, отменяет записи и получает
напоминания. Платформа — **нативное/гибридное мобильное приложение для iOS и Android**
(mobile-first для использования на берегу), распространяется через App Store и Google Play
(R-002). Веб-версии в скоупе нет.

**Скоуп — только роль «Клиент».** Инструктор и владелец работают через существующую
инфраструктуру/админку и в приложение не входят. Справочные данные — read-only из API;
оплата — офлайн (наличные / перевод), приложение лишь показывает цену и фиксирует запись.

**Что входит в MVP (принятые решения):**
- **Профиль клиента (R-009):** просмотр и редактирование имени и телефона, выход из аккаунта,
  удаление аккаунта — экран [SCR-007](SCR-007-profile.md).
- **Цена и прокат (R-010):** прокат — **отдельный тариф**; итог записи =
  `цена за место × число мест + тариф проката × число прокатных досок`. **Своя доска —
  бесплатно.** Валюта — **рубли (RUB)**. Итоговую цену считает и возвращает сервер; приложение
  показывает её, не пересчитывая.
- **Место встречи и маршрут (R-011):** обязательное место встречи + карта с геометрией маршрута
  на [SCR-003](SCR-003-slot-card.md)/[SCR-006](SCR-006-booking-details.md); при недоступности
  карты — текстовый fallback (см. [00-foundations §4.5](00-foundations.md#45-карта-маршрута-статичный-превью-яндекскарт)).
- **Отмена (R-014):** только **целиком** (нельзя отменить часть мест из записи).

Подробнее о продукте, аудитории, дизайн-принципах и контексте использования —
[00-foundations.md §1–2](00-foundations.md#1-продукт-и-аудитория).

## Роли и их задачи

| Роль | Что делает в приложении |
| :-- | :-- |
| **Клиент** | Просматривает и фильтрует слоты, записывается (себя + гости), выбирает вариант доски, смотрит цену, отменяет записи, видит свои брони, получает напоминания. |
| Инструктор / Владелец | **Не входят в приложение** — работают через существующую инфраструктуру/админку (вне скоупа). |

## Сквозные правила (foundations)

Единые для всех экранов требования вынесены в **[00-foundations.md](00-foundations.md)**:
дизайн-принципы, структурные токены (контраст, тач-зоны, иерархия — без бренда), каркас и
навигация (таб-бар, bottom sheet, нижний CTA), сквозной паттерн состояний
(Loading / Content / Empty / Error), tone of voice и общая микрокопия, доступность,
напоминания, безопасность данных, глоссарий.

> **Объём визуальных требований.** Документы задают **функционально-структурные** требования
> (иерархия, компоненты, поведение, ограничения), без прескриптивных цветов/шрифтов — бренд
> в исходной аналитике не зафиксирован; визуальный язык определяет дизайнер.

## Пользовательские сценарии (user flows)

Полная карта навигации — в [фича-листе §3](../5-mobile-app-spec/feature-list.md). Ключевые пути:

- **Вход (трёхшаговый):** запуск → SCR-001 (телефон → код из SMS → имя для нового) → SCR-002.
- **Запись (UC-1):** SCR-002 → SCR-003 → SCR-004 → BS-002 → (SCR-005). ≤ 3 экранов до подтверждения (NFR-2). После **первой** записи в BS-002 запрашивается разрешение на push.
- **Фильтрация (UC-3):** SCR-002 → BS-001 → SCR-002 (с применёнными фильтрами; множественный выбор типа/инструктора).
- **Отмена (UC-2):** SCR-005 → SCR-006 → BS-003 → SCR-006 (с обновлённым статусом). Отмена — только целиком.
- **Профиль:** SCR-007 — просмотр/редактирование имени и телефона, выход, удаление аккаунта, справочные пункты; выход/удаление → SCR-001.
- **Карта маршрута:** SCR-003 / SCR-006 → BS-004 (интерактивная карта с маршрутом) → возврат на исходный экран.

> **Карта маршрута и место встречи.** На [SCR-003](SCR-003-slot-card.md) и [SCR-006](SCR-006-booking-details.md)
> маршрут показывается **картой Яндекс (статичный превью с выделенной линией маршрута)**, а
> **место встречи** — обязательный элемент (пин на карте + текстовый адрес). Тап по карте
> открывает шторку [BS-004 «Карта маршрута»](BS-004-route-map.md) с открытой интерактивной картой
> и отрисованным маршрутом. Правила — [00-foundations.md §4.5](00-foundations.md#45-карта-маршрута-статичный-превью-яндекскарт).

## Состояния экранов

Единый паттерн для всех экранов с запросами — Loading (скелетон/шиммер) → Content →
Empty (заглушка + подсказка) → Error (+ «Обновить»). Описан один раз в
[00-foundations.md §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана); специфичные
состояния (disabled CTA «Записаться» при 0 мест, бейдж «Поздняя отмена» и т.п.) — в
соответствующих экранных документах.

## Тон, бренд, ограничения

- **Тон:** простой, прямой, дружелюбный, без давления и штрафов; обращение на «вы» —
  [00-foundations.md §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).
- **Бренд:** не зафиксирован в аналитике — визуальный язык за дизайнером; обязательны лишь
  структурные ограничения (контраст, размеры, иерархия) — [§3](00-foundations.md#3-структурные-токены-без-бренда).
- **Доступность / mobile-first:** крупные тач-зоны, высокий контраст, читаемость на солнце —
  [§3, §7](00-foundations.md#7-доступность-nfr-1).
- **Что НЕ делаем на старте (Phase 2+):** онлайн-оплата, рейтинг инструктора, авто-отмена по
  погоде, программа лояльности, публичные отзывы, интерфейсы инструктора/владельца — см.
  [фича-лист §7](../5-mobile-app-spec/feature-list.md).
