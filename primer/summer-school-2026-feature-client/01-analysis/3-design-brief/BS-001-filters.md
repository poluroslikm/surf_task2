## 1. Назначение и контекст

Шторка фильтров позволяет клиенту **уточнить список слотов под свой запрос** — не листать весь
каталог предстоящих прогулок, а быстро сузить его до релевантных вариантов (нужная дата,
подходящий тип маршрута, только со свободными местами, конкретный инструктор).

Шторка вызывается с экрана [SCR-002 «Список слотов»](SCR-002-slot-list.md) и поверх него же
применяет результат. Она не выполняет запрос самостоятельно для отображения — итоговая
фильтрованная выдача (включая пустой результат) рендерится на SCR-002. BS-001 — это **форма
выбора параметров**, а не отдельный экран результатов.

Контекст использования — берег, солнце, спешка (см. [foundations §1](00-foundations.md#1-продукт-и-аудитория)):
выбор должен быть крупным, быстрым и выполнимым одним пальцем.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** «Хочу видеть только подходящие мне прогулки» — отфильтровать каталог,
  чтобы быстрее дойти до записи (US-3, NFR-2 «короткий путь к записи»).
- **Задача дизайна:** дать собрать комбинацию условий за минимум тапов, понятно показать, что
  выбрано, и вернуть на список одним действием «Применить».

---

## 3. Навигация

**Входящие переходы:**
- Тап по кнопке/иконке **«Фильтры»** на [SCR-002 «Список слотов»](SCR-002-slot-list.md) →
  шторка открывается снизу вверх поверх списка.

**Исходящие переходы:**
- **«Применить»** → шторка закрывается, возврат на [SCR-002](SCR-002-slot-list.md); список
  перезапрашивается/перефильтровывается с учётом выбранных условий; на SCR-002 отображается
  **индикатор активных фильтров**.
- **«Сбросить»** → все параметры возвращаются к значениям по умолчанию (см. §7); при последующем
  «Применить» SCR-002 показывает дефолтную выдачу (слоты на ближайшие 7 дней, `only_available=false`).
- **Закрытие** (swipe-to-close / тап по бэкдропу / кнопка закрытия) → возврат на
  [SCR-002](SCR-002-slot-list.md) **без изменения** ранее применённых фильтров.

**Свойства bottom sheet** (бэкдроп, грабер, swipe-to-close, анимация, высота по контенту,
скрытие таб-бара) — см. [foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003).

## 4. Контент-инвентарь / Параметры фильтров

Все параметры справочные и read-only приходят из API. Логика комбинирования: **«ИЛИ» внутри
группы с множественным выбором** (тип маршрута, инструктор) и **«И» между группами** — слот
попадает в выдачу, если по каждой заданной группе выполнено хотя бы одно из выбранных значений.
Пример: выбраны типы «Новичковый» + «Опытный» и инструкторы «Анна» + «Игорь» → слот подходит,
если (его тип — новичковый ИЛИ опытный) И (его инструктор — Анна ИЛИ Игорь).

| # | Параметр | Поле данных | Тип значения | Поведение |
|---|----------|-------------|--------------|-----------|
| 1 | **Дата / период старта** | `start_at` | Быстрый пресет **или** произвольный диапазон (от–до) | Пресеты «Сегодня» / «Эта неделя» / «Выходные» + произвольный диапазон; по умолчанию — ближайшие 7 дней (дефолт API; больший период — выбором диапазона). |
| 2 | **Тип маршрута** | `route.type` | **Множественный выбор** из «новичковый» / «опытный» | Чип-лист с мультивыбором: можно выбрать оба типа (ИЛИ внутри группы). По умолчанию — не выбрано (любой). |
| 3 | **Только со свободными местами** | `free_seats > 0` | Переключатель (вкл/выкл) | По умолчанию — **выкл** (соответствует дефолту SCR-002: показываются и заполненные с пометкой «Мест нет»). |
| 4 | **Инструктор** | `instructor_id` | **Множественный выбор** из справочника инструкторов | Чип-лист/мульти-селект: можно выбрать нескольких (ИЛИ внутри группы). По умолчанию — не выбрано (любой); справочник грузится из API. |

> Числовые лимиты и потолки в фильтрах не зашиваются — критерий «есть места» опирается на
> данные слота (`free_seats`), см. [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

## 5. Структура и иерархия

Шторка делится на три зоны: **хедер** (грабер + заголовок + «Сбросить»), скроллируемый
**контент** (группы фильтров), фиксированную **нижнюю кнопку** «Применить».

```
┌──────────────────────────────────────┐
│                ▭ грабер               │  ← swipe-to-close
├──────────────────────────────────────┤
│  Фильтры                   Сбросить ▷ │  ← хедер: заголовок + сброс
├──────────────────────────────────────┤
│  Дата старта                          │  ← группа 1 (пресеты + диапазон)
│  [Сегодня] [Эта неделя] [Выходные]    │     быстрые пресеты (один)
│  ┌──────────────┐  ┌──────────────┐   │
│  │ С: 15 июня   │  │ По: 22 июня  │   │     произвольный диапазон
│  └──────────────┘  └──────────────┘   │
│                                        │
│  Тип маршрута                          │  ← группа 2 (чипы, мультивыбор)
│  [✓ Новичковый]  [ Опытный ]           │
│                                        │
│  ┌──────────────────────────────────┐ │  ← группа 3 (переключатель, выкл по умолч.)
│  │ Только со свободными местами  [ ○]│ │
│  └──────────────────────────────────┘ │
│                                        │
│  Инструктор                            │  ← группа 4 (чипы, мультивыбор)
│  [✓ Анна] [ Игорь ] [ Мария ] …       │
│         (скролл при нехватке высоты)   │
├──────────────────────────────────────┤
│ ┌────────────────────────────────────┐│  ← фикс. нижний CTA
│ │            Применить               ││
│ └────────────────────────────────────┘│
└──────────────────────────────────────┘
            ░░░ бэкдроп (затемнение) ░░░
```

Порядок групп — от наиболее частого критерия (дата, тип) к уточняющим (места, инструктор).

## 6. Компоненты и поведение

**6.1 Выбор даты (`start_at`) — пресеты + диапазон**
- **Быстрые пресеты** (чипы): «Сегодня», «Эта неделя», «Выходные» — выбирается один; тап
  заполняет соответствующий диапазон.
- **Произвольный диапазон:** два поля «С» и «По»; тап открывает выбор даты (календарь / нативный
  date picker), удобный одним пальцем. Допускается выбор одной даты (С = По) — фильтр на день.
- Выбор пресета и ручной диапазон синхронизированы (ручная правка снимает пресет).
- Прошедшие даты недоступны для выбора (фильтр работает по предстоящим слотам).

**6.2 Тип маршрута (`route.type`) — множественный выбор**
- Набор **чипов с мультивыбором**: «Новичковый» / «Опытный»; можно выбрать оба (ИЛИ внутри группы).
- Состояния чипа: невыбран / выбран (дублируется не только цветом — формой/иконкой,
  [foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)).

**6.3 Переключатель «Только со свободными местами» (`free_seats > 0`)**
- Toggle с подписью; по умолчанию **выключен** (в выдаче и заполненные слоты с пометкой «Мест нет»).
- Состояние считывается без опоры на цвет (положение + текст).

**6.4 Выбор инструктора (`instructor_id`) — множественный выбор**
- **Чип-лист / мульти-селект** инструкторов; можно отметить несколько (ИЛИ внутри группы). Без
  выбора — «любой».
- Список — справочные данные из API; на время загрузки — состояние загрузки (см. §7). Загружается
  отдельным справочным эндпоинтом (`GET /instructors` или эквивалент — уточняется в
  [instructors/api.yaml](../api/instructors/api.yaml)).

**6.5 Кнопка «Применить»** (фиксированная, нижняя, во всю ширину)
- Состояния: активна (всегда доступна — даже при дефолтных значениях применяет/подтверждает
  выбор), нажата (feedback), фокус.
- Тап → закрытие шторки + применение фильтров на SCR-002 (см. §3).

**6.6 Кнопка/действие «Сбросить»** (в хедере)
- Состояния: активно (если хоть один параметр отличается от дефолта) / неактивно или скрыто
  (если всё в дефолте — нечего сбрасывать).
- Тап → мгновенно возвращает все контролы к значениям по умолчанию **внутри шторки** (выдача
  на SCR-002 обновится после «Применить»).

**6.7 Свойства шторки** (swipe-to-close, бэкдроп с закрытием по тапу вне, грабер, явная кнопка
закрытия, анимация) — берутся из
[foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003), здесь не переописываются.

## 7. Состояния экрана

Опирается на сквозной паттерн состояний из
[foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Фильтры по умолчанию** | Дата — без пресета и диапазона (дефолт API — ближайшие 7 дней); тип маршрута — ничего не выбрано (любой); «только со свободными местами» — **выкл**; инструктор — ничего не выбрано (любой). «Сбросить» неактивно/скрыто. |
| **Выбраны фильтры** | Контролы отражают выбор (в т.ч. несколько типов/инструкторов); «Сбросить» активно; «Применить» подтверждает выбор. |
| **Загрузка списка инструкторов** (опц.) | Поле инструктора показывает состояние загрузки (скелетон/индикатор) до получения справочника из API; остальные группы интерактивны. |
| **Ошибка загрузки справочника** (опц.) | Поле инструктора показывает неблокирующую ошибку с возможностью повтора; прочие фильтры остаются доступны. |

> **Пустой результат после применения** обрабатывается **на [SCR-002](SCR-002-slot-list.md)**
> как empty state (UC-3 A1/E1), а не внутри шторки — шторка к этому моменту уже закрыта.
> Микрокопия пустого состояния — см. §8.

## 8. Валидации и микрокопия

| Правило | Поведение / текст |
|---------|-------------------|
| **Корректность диапазона дат** | Должно выполняться «от ≤ до». При попытке выбрать «По» раньше «С» — поля не дают задать некорректный диапазон (например, выбор «С» позже «По» сдвигает «По», или показывается подсказка). Применение некорректного диапазона блокируется. |
| **Пустой результат после применения** (на SCR-002) | Подсказка изменить/сбросить фильтры: **«Нет слотов по условиям. Попробуйте изменить фильтры.»** Рядом — действие вернуться к фильтрам / сбросить. |
| **Тон** | Простой, без вины и давления ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)). |

Лейблы контролов: «Фильтры», «Дата старта» (поля «С» / «По»), «Тип маршрута»
(«Новичковый» / «Опытный»), «Только со свободными местами», «Инструктор» («Любой»),
«Применить», «Сбросить».

## 9. Доступность и mobile-first

Базовые требования — [foundations §3.1, §3.2, §7](00-foundations.md#7-доступность-nfr-1). Дополнительно для BS-001:
- **Крупные контролы:** чипы типа маршрута, toggle и поля дат — не меньше минимальной тач-зоны
  (≥ 44–48 pt), с запасом отступа от соседей («мокрый палец»).
- **Выбор дат одним пальцем:** компонент даты удобен для управления большим пальцем одной руки;
  целевые области дней/кнопок календаря крупные.
- Состояния выбора (чип, toggle) не передаются только цветом — дублируются формой/иконкой/текстом.
- Все контролы имеют доступные имена; кнопка «Применить» достижима без скролла (фиксирована внизу).

## 10. Критерии приёмки дизайна

```gherkin
Функция: Шторка фильтров BS-001

  Сценарий: AC-001 — Открытие шторки с экрана списка
    Дано клиент на экране SCR-002 «Список слотов»
    Когда он нажимает «Фильтры»
    Тогда снизу открывается шторка BS-001 с грабером, заголовком, группами фильтров и кнопкой «Применить»
    И значения отражают дефолт: даты — без пресета и диапазона (дефолт API — ближайшие 7 дней), тип — любой (ничего не выбрано), «только со свободными местами» — выкл, инструктор — любой (ничего не выбрано)

  Сценарий: AC-002 — Комбинирование групп по «И», значений в группе по «ИЛИ»
    Дано открыта шторка BS-001
    Когда клиент задаёт диапазон дат, отмечает типы «Новичковый» и «Опытный» и двух инструкторов
    И нажимает «Применить»
    Тогда на SCR-002 остаются слоты, удовлетворяющие ВСЕМ группам (AND),
      где внутри группы достаточно совпадения с любым из выбранных значений (OR)

  Сценарий: AC-003 — Применение возвращает на список с индикатором
    Дано выбран хотя бы один фильтр, отличный от дефолта
    Когда клиент нажимает «Применить»
    Тогда шторка закрывается и происходит возврат на SCR-002
    И на SCR-002 виден индикатор активных фильтров

  Сценарий: AC-004 — Сброс возвращает к дефолтной выдаче (ближайшие 7 дней)
    Дано заданы какие-либо фильтры
    Когда клиент нажимает «Сбросить» и затем «Применить»
    Тогда все параметры возвращаются к значениям по умолчанию
    И SCR-002 показывает дефолтную выдачу (слоты на ближайшие 7 дней, `only_available=false`)
    И индикатор активных фильтров на SCR-002 снят

  Сценарий: AC-005 — Закрытие без применения сохраняет прежние фильтры
    Дано на SCR-002 ранее были применены фильтры
    Когда клиент открывает шторку, меняет значения и закрывает её свайпом вниз или тапом по бэкдропу
    Тогда ранее применённые фильтры на SCR-002 не изменяются

  Сценарий: AC-006 — Корректность диапазона дат
    Дано открыта группа «Дата старта»
    Когда клиент пытается задать дату «По» раньше даты «С»
    Тогда некорректный диапазон не применяется (поля не дают его задать либо показывают подсказку)

  Сценарий: AC-007 — Пустой результат после применения (UC-3 A1/E1)
    Дано клиент применил фильтры, под которые нет ни одного слота
    Когда происходит возврат на SCR-002
    Тогда показывается empty state с текстом «Нет слотов по условиям. Попробуйте изменить фильтры.»
    И доступно действие изменить/сбросить фильтры
```

## 11. Решения и параметры конфигурации

| # | Решение |
|---|---------|
| 1 | **Дата:** быстрые пресеты «Сегодня» / «Эта неделя» / «Выходные» + произвольный диапазон (§6.1). |
| 2 | **Тип маршрута — множественный выбор** (чипы): можно выбрать оба типа; ИЛИ внутри группы (§6.2). |
| 3 | **Инструктор — множественный выбор** (чипы/мульти-селект): можно выбрать нескольких; ИЛИ внутри группы (§6.4). |
| 4 | **Живой счётчик результатов в шторке — не делаем.** Результат и пустое состояние видны на [SCR-002](SCR-002-slot-list.md) после «Применить». |
| 5 | *(Параметр API, не блокер)* **Справочник инструкторов** грузится отдельным эндпоинтом (`GET /instructors` или эквивалент) — финализируется в [instructors/api.yaml](../api/instructors/api.yaml). |
