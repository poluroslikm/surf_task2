## 1. Назначение и контекст

**SCR-002 «Список слотов» — главный экран приложения** и стартовая вкладка авторизованной
зоны. Показывает каталог доступных групповых SUP-прогулок и служит **единственной точкой
входа в сценарий записи**.

По умолчанию отображаются слоты на **ближайшие 7 дней** (дефолт API; больший период — фильтром
дат), отсортированные по ближайшей дате старта
и сгруппированные по дням; заполненные слоты показываются с пометкой **«Мест нет»** (см. §6.5).
Фильтр «только со свободными местами» доступен в [BS-001](BS-001-filters.md), но **по умолчанию
выключен**. Отсюда клиент открывает карточку слота для записи или уточняет выдачу через фильтры.

Экран работает с API в режиме read-only (см. [`00-foundations.md` §1](00-foundations.md)) и
подчиняется сквозному паттерну состояний ([`00-foundations.md` §5](00-foundations.md)).

---

## 2. Пользователь и цель

**Роль — Клиент.** Контекст использования — берег, нередко яркое солнце, мокрые/холодные руки,
спешка перед выходом на воду (см. [`00-foundations.md` §1](00-foundations.md)).

**Цель на экране:** быстро просмотреть ближайшие прогулки и за минимум шагов выбрать подходящую
(дата, тип маршрута, инструктор, цена, есть ли места). Список оптимизирован под **быстрый выбор**:
крупные карточки, контрастные ключевые числа, минимум мелкого текста (принципы P1, P2, P4 в
[`00-foundations.md` §2](00-foundations.md)).

## 3. Навигация

### Входящие переходы
- После успешного входа/регистрации с [SCR-001](SCR-001-registration.md) — это стартовый экран АЗ.
- По тапу на таб **«Прогулки»** в таб-баре (с любого корневого экрана вкладок).
- Возврат с карточки слота [SCR-003](SCR-003-slot-card.md) (кнопка «назад») и из сценария записи.
- Закрытие шторки фильтров [BS-001](BS-001-filters.md) (Применить / Сбросить / закрытие).

### Исходящие переходы
- Тап по карточке слота → карточка слота [SCR-003](SCR-003-slot-card.md).
- Тап по кнопке/иконке **«Фильтры»** → шторка [BS-001](BS-001-filters.md).
- Таб **«Мои записи»** → [SCR-005](SCR-005-my-bookings.md).
- Таб **«Профиль»** → [SCR-007](SCR-007-profile.md).

Таб-бар (три раздела, подписи под иконками, поведение на вложенных экранах) описан в
[`00-foundations.md` §4.2](00-foundations.md) — **ссылаемся, не переописываем**.

## 4. Контент-инвентарь

Каждая **карточка слота** агрегирует данные слота, маршрута и инструктора
(см. [модель данных](../4-design/data-model.md)):

| Элемент | Источник данных | Назначение в UI |
|---------|------------------|------------------|
| Дата и время старта | `slot.start_at` | Когда прогулка; ключевое число (контрастно). |
| Название маршрута | `route.name` | Что за прогулка. |
| Тип маршрута | `route.type` (новичковый / опытный) | Подходит ли по уровню; читаемый лейбл, не только цвет. |
| Инструктор | `instructor.name` | Кто ведёт. |
| Цена | `slot.price` | Стоимость места; ключевое число (контрастно). |
| Всего мест | `slot.total_seats` | Размер группы (M в формулировке «N из M»). |
| Свободно мест | `slot.free_seats` | Доступность; ключевое число (контрастно). |

> **Без хардкода лимитов.** Свободные места — данные слота; по сквозному правилу
> ([`00-foundations.md` §6](00-foundations.md)) доступность мест считается как
> `min(free_seats, route.capacity_cap, 3)`. Прокатный фонд — **отдельная проверка**
> `rental_count ≤ free_rental_boards`. Конкретные числа подставляются из данных,
> в макет не зашиваются.

**Уровень экрана (не карточки):**
- Заголовок экрана.
- Кнопка/иконка **«Фильтры»** + **индикатор активных фильтров** (видим, когда применён хотя бы
  один фильтр; см. [`00-foundations.md` §3.5](00-foundations.md)).

## 5. Структура и иерархия

Каркас следует базовому ([`00-foundations.md` §4.1](00-foundations.md)): фиксированный хедер →
скролл-список → таб-бар. Нижнего CTA на этом экране нет.

```
┌─────────────────────────────────┐
│ Прогулки         [⚙ Фильтры •]  │  ← хедер: заголовок + кнопка фильтров
│                          ▲ индикатор активных фильтров
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ [карточка слота — см. ниже] │ │
│ └─────────────────────────────┘ │  ← скролл-список карточек
│ ┌─────────────────────────────┐ │
│ │ [карточка слота]            │ │
│ └─────────────────────────────┘ │
│              ⋮ (скролл)          │
├─────────────────────────────────┤
│ [Прогулки] [Мои записи] [Профиль]│  ← таб-бар (foundations §4.2)
└─────────────────────────────────┘
```

**Карточка слота (вся область — одна тап-зона):**

```
┌───────────────────────────────────────┐
│  Сб, 21 июня · 10:00                    │  ← start_at (крупно, контрастно)
│  Утренний залив · Новичковый            │  ← route.name · route.type
│  Инструктор: Анна                       │  ← instructor.name
│  ─────────────────────────────────────  │
│  1500 ₽            Свободно 4 из 8       │  ← price | free_seats из total_seats
└───────────────────────────────────────┘
```

Иерархия в карточке: дата/время старта (заголовок карточки) → маршрут+тип → инструктор →
строка «цена / свободно мест». Уровни — по [`00-foundations.md` §3.3](00-foundations.md).
Значения-примеры в wireframe условны.

## 6. Компоненты и поведение

### 6.1 Карточка слота
- **Вся карточка — единая тап-зона** (≥ минимальной тач-зоны, [`00-foundations.md` §3.1](00-foundations.md)),
  по тапу — переход в [SCR-003](SCR-003-slot-card.md). Внутри карточки нет конкурирующих кликов.
- Видимая обратная связь на тап (press-state), фокус для screen reader
  ([`00-foundations.md` §7](00-foundations.md)).
- Карточки разделены явными отступами/границами ([`00-foundations.md` §3.4](00-foundations.md)).

### 6.2 Кнопка/иконка «Фильтры»
- Размещена в хедере; открывает [BS-001](BS-001-filters.md).
- **Индикатор активных фильтров** — виден, когда применён ≥ 1 фильтр (точка/бейдж/счётчик);
  считывается с первого взгляда и дублируется не только цветом ([`00-foundations.md` §3.5](00-foundations.md)).
- Тач-зона ≥ минимальной; подпись/доступное имя обязательны.

### 6.3 Сортировка и группировка
- Прогулки сортируются **по ближайшей дате/времени старта** (раньше — выше).
- При длинном списке — **заголовки-секции по дням** (sticky-разделители «дата»), чтобы клиент
  ориентировался в расписании. Заголовок дня — не только цвет, но текст.

### 6.4 Обновление доступности (pull-to-refresh)
- Жест «потянуть вниз» обновляет список и **актуальную доступность мест**. Во время обновления —
  индикатор обновления; контент не очищается без необходимости (воспринимаемая скорость, P4).

### 6.5 Подгрузка списка
- В MVP список предстоящих слотов **грузится целиком** (горизонт расписания ограничен — неделя
  вперёд, см. [domain](../1-elicitation/domain-description.md)); специальная пагинация не нужна.
- *(Задел, не блокер MVP.)* Если объём вырастет, на API закладывается **курсорная пагинация** с
  догрузкой по достижению конца списка и скелетоном в подвале. Включается без изменения макета.

### 6.6 Слот без свободных мест
- Слот с `free_seats = 0` **показывается** с явной пометкой **«Мест нет»** (текст + иконка/форма,
  не только цвет) и **некликабелен** — не открывает сценарий записи.
- Скрыть заполненные слоты можно фильтром «только со свободными местами» в
  [BS-001](BS-001-filters.md) (по умолчанию **выключен**).

## 7. Состояния экрана

Применяется сквозной паттерн ([`00-foundations.md` §5](00-foundations.md)). Специфика экрана:

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетоны карточек слотов (форма будущих карточек), не пустой экран. |
| **Content** | Список карточек предстоящих слотов (со свободными и заполненными — последние с пометкой «Мест нет»), сортировка по ближайшей дате, секции по дням. |
| **Empty (нет слотов)** | Заглушка: «Пока нет доступных прогулок» + подсказка зайти позже / обновить. |
| **Empty (нет слотов по фильтрам)** | Заглушка с подсказкой **изменить или сбросить фильтры** + быстрый доступ к [BS-001](BS-001-filters.md) (UC-3 A1/E1). |
| **Error** | Заглушка ошибки + кнопка **«Обновить»** (общий текст сетевой ошибки — [`00-foundations.md` §6](00-foundations.md)). |

**Скорость отклика:** загрузка/обновление списка ощущается **< 2–3 с** (NFR-6, принцип P4);
до получения данных показываются скелетоны, а не пустой экран.

## 8. Валидации и микрокопия

- **Счётчик свободных мест отражает актуальные данные слота** — обновляется по
  pull-to-refresh и при возврате на экран; не кэшируется в устаревшем виде.
- Формулировка доступности — **«Свободно N из M»**, где N = `free_seats`, M = `total_seats`
  (числа из данных, не зашиты).
- Слот без мест — пометка **«Мест нет»** (текст + иконка/форма, не только цвет).
- Лейбл типа маршрута — **«Новичковый» / «Опытный»** (из `route.type`).
- Цена и время — из данных слота; формат единый по приложению.
- Тексты Empty:
  - нет слотов вообще — «Пока нет доступных прогулок. Загляните позже.»;
  - нет по фильтрам — «По выбранным фильтрам ничего не нашлось. Измените или сбросьте фильтры.»
- Тон — простой и без давления ([`00-foundations.md` §6](00-foundations.md)).

---

## 9. Доступность и mobile-first

- Все сквозные правила — [`00-foundations.md` §7](00-foundations.md) и §3.1–3.2.
- **Крупные карточки** и единая тап-зона на карточку — удобно «мокрым пальцем» на берегу.
- **Ключевые числа контрастны и крупны**: свободно мест, цена, время старта (§3.2 foundations).
- Тип маршрута, индикатор фильтров и пометка «Мест нет» — не только цветом, но текстом/иконкой.
- Поддержка системного увеличения шрифта: карточка не «ломается» при крупном тексте.
- Карточки и кнопка «Фильтры» имеют доступные имена для screen reader.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: По умолчанию — предстоящие слоты со свободными местами
  Дано клиент авторизован и открыл вкладку «Прогулки»
  И фильтры не применены
  Когда список загружен
  Тогда показаны только предстоящие слоты со свободными местами
  И каждая карточка содержит дату/время старта, маршрут и его тип,
    инструктора, цену и «Свободно N из M»

AC-002: Переход в карточку слота по тапу
  Дано на экране есть карточка слота
  Когда клиент тапает в любую область карточки
  Тогда открывается карточка слота SCR-003 для этого слота

AC-003: Открытие фильтров и индикатор активных фильтров
  Дано клиент на экране списка слотов
  Когда клиент тапает «Фильтры»
  Тогда открывается шторка BS-001
  И после применения хотя бы одного фильтра в хедере виден индикатор активных фильтров

AC-004: Пустой результат по фильтрам
  Дано применены фильтры
  Когда по ним нет ни одного слота
  Тогда показан empty state с подсказкой изменить или сбросить фильтры
  И доступен быстрый переход к фильтрам

AC-005: Обновление по pull-to-refresh
  Дано список слотов отображается
  Когда клиент тянет список вниз
  Тогда список и счётчики свободных мест обновляются актуальными данными
  И во время обновления виден индикатор обновления

AC-006: Состояние загрузки — скелетоны
  Дано экран открывается и данные ещё не получены
  Когда идёт первичная загрузка
  Тогда показаны скелетоны карточек, а не пустой экран
  И отклик ощущается в пределах 2–3 секунд (NFR-6)

AC-007: Ошибка загрузки
  Дано запрос списка завершился ошибкой
  Когда отображается экран
  Тогда показана заглушка ошибки с кнопкой «Обновить»
  И повтор по «Обновить» возвращает экран в загрузку

AC-008: Слот без свободных мест
  Дано у слота нет свободных мест
  И фильтр «только со свободными местами» выключен (по умолчанию)
  Тогда слот показан в списке с пометкой «Мест нет» и некликабелен
  И при включении фильтра «только со свободными местами» такой слот скрывается

AC-009: Сортировка и группировка по дням
  Дано список содержит слоты на разные даты
  Когда список загружен
  Тогда слоты отсортированы по ближайшей дате/времени старта
  И при длинном списке показаны заголовки-секции по дням
```

## 11. Решения и параметры конфигурации

| # | Решение |
|---|---------|
| 1 | **Сортировка по умолчанию** — по ближайшей дате/времени старта; заголовки-секции по дням при длинном списке (§6.3). |
| 2 | **Заполненные слоты** — показываются с пометкой «Мест нет» и некликабельны; скрыть можно фильтром «только со свободными» (по умолчанию выключен) (§6.6). |
| 3 | **Пагинация** — в MVP список грузится целиком (горизонт ~неделя); курсорная пагинация заложена на будущее как задел, *не блокер* (§6.5). |
