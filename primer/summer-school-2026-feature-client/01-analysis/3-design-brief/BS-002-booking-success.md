## 1. Назначение и контекст

Шторка завершает основной сценарий записи: **подтверждает, что бронь успешно создана**, даёт
клиенту наглядную сводку того, на что он записался, напоминает об **офлайн-оплате** и
предлагает дальнейший шаг — посмотреть запись в «Моих бронированиях» или вернуться к слотам.

BS-002 — это **финальный, успешный** результат потока записи. Она открывается **после
успешного подтверждения** на экране оформления [SCR-004](SCR-004-booking.md). Ошибки записи
(нехватка мест, нехватка прокатных досок, гонка запросов, сетевой сбой) обрабатываются **на
SCR-004** и до этой шторки не доходят — см. [UC-1](../2-requirements/use-cases.md). Таким
образом, BS-002 закрывает путь записи в рамках принципа «короткий путь к записи»
(см. [foundations §2, P2](00-foundations.md#2-дизайн-принципы)).

---

## 2. Пользователь и цель

- **Пользователь:** клиент, только что оформивший запись (роль — единственная, см.
  [foundations §1](00-foundations.md#1-продукт-и-аудитория)). Часто стоит на берегу, спешит,
  хочет убедиться, что «всё получилось», и понять, что делать дальше.
- **Цель пользователя:** получить однозначное подтверждение успеха, проверить ключевые
  параметры брони (когда, какой маршрут, сколько мест, сколько платить) и понять способ оплаты.
- **Цель продукта:** закрыть петлю записи спокойным позитивным завершением (без давления),
  снизить тревогу «записался ли я», мягко подвести к разделу «Мои бронирования»
  ([US-5](../2-requirements/user-stories.md)).

## 3. Навигация

### Входящие переходы
- **SCR-004 → BS-002** — после успешного подтверждения записи на экране
  [Оформление записи](SCR-004-booking.md) (бронь создана на бэкенде, места уменьшены).
  Это единственная точка входа.

### Исходящие переходы
- **«Мои бронирования» → [SCR-005](SCR-005-my-bookings.md)** — переход к списку своих записей;
  созданная бронь видна там.
- **«Готово» / закрытие → возврат к списку слотов [SCR-002](SCR-002-slot-list.md)** — шторка
  закрывается, клиент оказывается на корневой вкладке «Прогулки» (стек оформления записи
  завершён). Промежуточные экраны [SCR-003](SCR-003-slot-card.md)/[SCR-004](SCR-004-booking.md)
  в стеке не остаются — путь записи окончен.

### Свойства шторки
Единые правила bottom sheet (высота по контенту, бэкдроп, грабер, swipe-to-close, анимация,
расположение кнопок) — см. [foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003).
Специфика BS-002 — в §6.

## 4. Контент-инвентарь

Все значения подставляются из данных созданной брони/слота; числа и лимиты **не хардкодятся**
(см. [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).

| # | Элемент | Источник данных | Назначение |
|---|---------|-----------------|------------|
| 1 | Иконка успеха | — | Невербальный сигнал «готово» (не только цветом, см. §9). |
| 2 | Заголовок «Вы записаны» (формулировку уточняет дизайнер/копирайтер) | — | Однозначное подтверждение. |
| 3 | Дата и время старта | `slot.start_at` | Когда прогулка. |
| 4 | Маршрут | `slot.route.name` | На какой маршрут запись. |
| 5 | Инструктор | `slot.instructor.name` | Кто ведёт. |
| 6 | Число мест | `booking.seats_count` | Сколько мест забронировано (себя + гости). |
| 7 | Вариант досок | `own` / `rental` по местам, `rental_count` | Сколько своих / прокатных досок (лейблы — из foundations §6). |
| 8 | Итоговая цена | расчёт по слоту/брони | Сколько платить. |
| 9 | Напоминание об оплате | текст из foundations §6 | Способ оплаты — офлайн. |
| 10 | Кнопка «Мои бронирования» | — | Переход к [SCR-005](SCR-005-my-bookings.md). |
| 11 | Кнопка «Готово» / закрытие | — | Закрыть шторку → [SCR-002](SCR-002-slot-list.md). |

> **Вариант досок** отображается так, чтобы было ясно соотношение своих и прокатных мест
> (например, «N мест · из них M прокатных»). Точная компоновка — за дизайнером; лейблы
> «Своя доска» / «Прокатная доска» — из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

## 5. Структура и иерархия

Сверху вниз:

1. **Грабер** (полоска для swipe-to-close).
2. **Блок успеха:** иконка успеха + заголовок-подтверждение.
3. **Сводка брони:** дата/время старта, маршрут, инструктор, число мест, вариант досок,
   итоговая цена.
4. **Напоминание об оплате** (офлайн: наличные / перевод).
5. **Кнопки действий** (внизу шторки): основная — **«Мои бронирования»**, вторичная —
   **«Готово»** (закрытие).

### ASCII-wireframe

```
┌─────────────────────────────────────┐
│                ▭▭▭                    │  ← грабер (swipe-to-close)
│                                       │
│               ( ✓ )                   │  ← иконка успеха (форма + текст, не только цвет)
│            Вы записаны                │  ← заголовок-подтверждение
│                                       │
│  ┌─────────────────────────────────┐ │
│  │ 🗓  <start_at: дата · время>     │ │
│  │ 〰  Маршрут: <route.name>        │ │  ← сводка брони
│  │ 👤  Инструктор: <instructor.name>│ │
│  │ 👥  Мест: <seats_count>          │ │
│  │ 🛶  Доски: своих <…> · прокат <…>│ │  ← own / rental_count
│  │ ───────────────────────────────  │ │
│  │ 💳  Итого: <итоговая цена>        │ │  ← крупно, контрастно
│  └─────────────────────────────────┘ │
│                                       │
│  ⓘ Оплата на месте: наличные         │  ← напоминание об офлайн-оплате
│     или перевод на карту.             │
│                                       │
│  ┌─────────────────────────────────┐ │
│  │      Мои бронирования           │ │  ← основная кнопка → SCR-005
│  └─────────────────────────────────┘ │
│  ┌─────────────────────────────────┐ │
│  │           Готово                │ │  ← вторичная кнопка → закрыть → SCR-002
│  └─────────────────────────────────┘ │
└─────────────────────────────────────┘
        ░░░ бэкдроп (затемнение) ░░░
```

> Иконки в вайрфрейме — условные, иллюстрируют состав сводки; конкретный набор и стиль —
> за дизайнером. Ключевые числа (старт, мест, цена) — крупные и контрастные
> ([foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)).

## 6. Компоненты и поведение

### 6.1 Блок успеха
- Иконка успеха + короткий заголовок. Состояние «успех» передаётся **формой/иконкой и
  текстом**, а не только цветом ([foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)).
- Тон — спокойный, дружелюбный, без восклицательного давления
  ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).

### 6.2 Сводка брони
- Карточка/список «лейбл → значение» с параметрами из §4. Один столбец (mobile-first).
- Итоговая цена выделена как самое важное число блока.
- Вариант досок показывает соотношение своих/прокатных; при `rental_count = 0` строка про
  прокат может скрываться (за дизайнером), но факт «своя доска» остаётся понятен.

### 6.3 Кнопки
- **«Мои бронирования»** — основной CTA, во всю ширину контентной области, ведёт на
  [SCR-005](SCR-005-my-bookings.md).
- **«Готово»** — вторичная кнопка, закрывает шторку и возвращает на
  [SCR-002](SCR-002-slot-list.md).
- Размеры и отступы тач-зон — крупные, по
  [foundations §3.1](00-foundations.md#31-тач-зоны-и-размеры) (важно для «мокрого пальца» на берегу).

### 6.4 Запрос разрешения на push (после первой записи)
- **Только при первой успешной записи** клиента: после показа сводки предлагается включить
  напоминания о прогулке — затем вызывается **системный запрос разрешения на push**
  ([foundations §8.1](00-foundations.md#81-напоминания-и-уведомления-fr-33-nfr-13)).
- Это **единственное место** запроса разрешения в продукте; на [SCR-001](SCR-001-registration.md)
  и в [SCR-007](SCR-007-profile.md) он не запрашивается.
- При последующих записях разрешение **повторно не запрашивается**. Отказ не блокирует запись и
  не повторяется навязчиво (P6 — без давления). Спокойная подводка к ценности («напомним за
  N часов до старта»); конкретный текст — за копирайтером.

### 6.5 Поведение bottom sheet
- Высота по контенту, бэкдроп, грабер, swipe-to-close вниз, плавная анимация — по
  [foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003).
- Закрытие тапом по бэкдропу, свайпом вниз или кнопкой «Готово» — все ведут к одному исходу
  (возврат к [SCR-002](SCR-002-slot-list.md)); бронь уже сохранена, повторное действие не
  требуется. BS-002 — **не критичное подтверждение**, поэтому закрытие свободным жестом
  допустимо.
- Длинная сводка (например, при максимуме мест) скроллится внутри шторки; кнопки остаются
  достижимы.

## 7. Состояния экрана

BS-002 открывается **только по факту уже успешно созданной брони**, поэтому её основное и
практически единственное состояние — **Content (успех)**.

| Состояние | Применимость | Примечание |
|-----------|--------------|------------|
| **Content (успех)** | Да | Сводка созданной брони + дальнейшие шаги. Основной и целевой режим. |
| **Loading** | Нет (в шторке) | Запрос на создание брони и его индикация выполняются на [SCR-004](SCR-004-booking.md) до открытия шторки. |
| **Empty** | Нет | Не бывает: шторка появляется только при наличии созданной брони. |
| **Error** | Нет (в шторке) | Все ошибки записи обрабатываются на [SCR-004](SCR-004-booking.md) — см. [UC-1, E1–E4](../2-requirements/use-cases.md). До BS-002 ошибочный сценарий не доходит. |

Общий паттерн состояний — см.
[foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).

## 8. Валидации и микрокопия

BS-002 не содержит вводимых пользователем полей, поэтому валидаций ввода нет. Контентные и
поведенческие правила:

| Правило | Описание |
|---------|----------|
| Напоминание об оплате | Текст из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия): **«Оплата на месте: наличные или перевод на карту.»** Не дублировать другими формулировками. |
| Лейблы досок | «Своя доска» / «Прокатная доска» — из foundations §6. |
| Запись отражена | После закрытия шторки созданная бронь должна присутствовать в «Моих бронированиях» [SCR-005](SCR-005-my-bookings.md) (FR-30 / US-5). |
| Места уменьшены | Свободные места слота уменьшены на число занятых мест; прокатный фонд уменьшен только на `rental_count` («свои» доски фонд не уменьшают). Данные на [SCR-002](SCR-002-slot-list.md)/[SCR-003](SCR-003-slot-card.md) при следующем обновлении отражают новую доступность. |
| Тон | Без давления и вины, обращение на «вы» (foundations §6). |
| Числа | Цена, число мест, время старта подставляются из данных — без хардкода (foundations §6). |

## 9. Доступность и mobile-first

Базовые требования — см.
[foundations §3 и §7](00-foundations.md#7-доступность-nfr-1). Специфика BS-002:

- **Крупные кнопки** «Мои бронирования» и «Готово» — тач-зоны по
  [§3.1](00-foundations.md#31-тач-зоны-и-размеры); легко нажимать мокрыми/холодными руками.
- **Понятная иконка успеха** — смысл «готово» считывается **формой и текстом**, а не только
  цветом ([§3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)); читаемо на ярком солнце.
- Ключевые числа сводки (старт, мест, цена) — крупные и контрастные.
- Все элементы (кнопки, грабер, иконка успеха) имеют доступные имена для screen reader; о
  факте успеха сообщается текстом, а не только иконкой.
- Layout не «ломается» при системном увеличении шрифта; при необходимости содержимое
  скроллится внутри шторки.

## 10. Критерии приёмки дизайна

```gherkin
Функция: Подтверждение успешной записи (BS-002)

  Сценарий: AC-001 — после успешной записи показана сводка
    Дано клиент подтвердил запись на экране SCR-004
    И бронь успешно создана
    Когда открывается шторка BS-002
    Тогда виден блок успеха (иконка + заголокок-подтверждение)
    И видна сводка брони: дата/время старта, маршрут, инструктор, число мест, вариант досок и итоговая цена
    И все значения подставлены из данных брони, без хардкода

  Сценарий: AC-002 — видно напоминание об оплате
    Дано открыта шторка BS-002
    Тогда показано напоминание «Оплата на месте: наличные или перевод на карту.»

  Сценарий: AC-003 — переход в «Мои бронирования»
    Дано открыта шторка BS-002
    Когда клиент нажимает «Мои бронирования»
    Тогда выполняется переход на экран SCR-005

  Сценарий: AC-004 — запись отражена в списке
    Дано бронь успешно создана через BS-002
    Когда клиент открывает «Мои бронирования» (SCR-005)
    Тогда созданная запись присутствует в списке
    И свободные места соответствующего слота уменьшены

  Сценарий: AC-005 — закрытие возвращает к списку слотов
    Дано открыта шторка BS-002
    Когда клиент нажимает «Готово» (или закрывает шторку свайпом/тапом по бэкдропу)
    Тогда шторка закрывается
    И клиент оказывается на списке слотов SCR-002

  Сценарий: AC-006 — состав вариантов досок
    Дано в брони есть прокатные места (rental_count > 0)
    Когда открыта шторка BS-002
    Тогда сводка показывает соотношение своих и прокатных досок
    И прокатный фонд слота уменьшен только на rental_count

  Сценарий: AC-007 — запрос push после первой записи
    Дано это первая успешная запись клиента
    Когда открыта шторка BS-002
    Тогда после сводки предлагается включить напоминания и показывается системный запрос разрешения на push
    И при последующих записях этот запрос не повторяется
    И отказ не блокирует запись
```

## 11. Решения

| # | Решение |
|---|---------|
| 1 | **«Поделиться» / «Добавить в календарь» — не делаем** в MVP (добавляет шаги, противоречит P2). |
| 2 | **Автозакрытие по таймеру — нет.** Шторка закрывается только по действию пользователя (чтобы успел прочитать сводку и напоминание об оплате). |
| 3 | **Разрешение на push запрашивается здесь** — только после **первой** успешной записи (§6.4, [foundations §8.1](00-foundations.md#81-напоминания-и-уведомления-fr-33-nfr-13)). |
| 4 | **Правило отмены на подтверждении не дублируем** — оно раскрыто на [SCR-006](SCR-006-booking-details.md) / [BS-003](BS-003-cancel-confirm.md) и в [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия). |
